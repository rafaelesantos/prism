# What is Prism?

Prism is a **full-stack Swift ecosystem** built entirely on Apple frameworks — zero third-party dependencies. It spans **10 modules** covering everything from server-side HTTP to client-side UI components, state management, networking, ML/AI, and native platform capabilities.

<CardGroup cols={2}>
  <Card title="Server" icon="server">
    **PrismServer** — HTTP server, routing, database, GraphQL, MCP, WebSocket rooms, middleware, cron, and more.
  </Card>
  <Card title="Client" icon="mobile">
    **PrismUI** — 100+ SwiftUI components, design tokens, theming, charts, forms, accessibility, and dev tools.
  </Card>
  <Card title="Architecture" icon="sitemap">
    **PrismArchitecture** — composable state management with stores, reducers, effects, middleware, and navigation.
  </Card>
  <Card title="Intelligence" icon="brain">
    **PrismIntelligence** — on-device ML, Apple Intelligence, NLP, RAG pipelines, and structured output parsing.
  </Card>
</CardGroup>

```swift title="Hello World — Server"
import PrismServer

let server = PrismHTTPServer(port: 8080)

await server.get("/") { request in
    .json(["message": "Hello, Prism!"])
}

await server.get("/users/:id") { request in
    let id = request.parameters["id"]!
    .json(["id": id, "name": "Alice"])
}

try await server.start()
```

```swift title="Hello World — SwiftUI App"
import Prism

struct CounterFeature: PrismReducer {
    struct State: Sendable, Equatable { var count = 0 }
    enum Action: Sendable { case increment, decrement }

    func reduce(into state: inout State, action: Action) -> PrismEffect<Action> {
        switch action {
        case .increment: state.count += 1
        case .decrement: state.count -= 1
        }
        return .none
    }
}

struct CounterView: View {
    let store: PrismStore<CounterFeature.State, CounterFeature.Action>

    var body: some View {
        VStack {
            Text("\(store.state.count)")
                .prismTypography(.headlineLarge)
            HStack {
                PrismButton("−", variant: .outlined) { store.send(.decrement) }
                PrismButton("+", variant: .filled) { store.send(.increment) }
            }
        }
    }
}
```

No configuration files, no boilerplate, no dependency resolution. Just Swift.

## The Prism Ecosystem

Prism is organized into **10 focused modules** that work together or independently:

<AccordionGroup>
  <Accordion title="PrismServer — Server-Side Framework">
    Full HTTP/1.1 server, routing, middleware (CORS, auth, rate limiting, compression), SQLite database with ORM, WebSocket rooms, GraphQL engine, MCP server, SSE, cron jobs, background tasks, cluster mode, and more. 36+ server modules.
  </Accordion>
  <Accordion title="PrismArchitecture — State Management">
    Composable architecture inspired by TCA. PrismStore for observable state, PrismReducer for pure business logic, PrismEffect for async side effects, PrismMiddleware for cross-cutting concerns, PrismRouter for type-safe navigation. Plus: state persistence, time-travel debugging, undo/redo, derived stores, and testing utilities.
  </Accordion>
  <Accordion title="PrismUI — Design System & Components">
    100+ SwiftUI components with semantic design tokens. Theming (dark, high-contrast, brand), primitives (buttons, cards, chips), forms with validation, charts (bar, line, radar, heatmap), dashboard widgets, chat components, rich text editors, accessibility tools, animations, and developer tools like Storybook and snapshot testing.
  </Accordion>
  <Accordion title="PrismNetwork — HTTP Client">
    Type-safe HTTP client with endpoint protocols, response caching (LRU with TTL), retry policies (exponential backoff, linear), request deduplication, offline queue with auto-flush, multipart uploads, GraphQL client, and WebSocket client.
  </Accordion>
  <Accordion title="PrismFoundation — Core Utilities">
    Base protocols (PrismEntity, PrismError, PrismMock), structured logging, performance tracing, crash reporting, analytics funnels, UserDefaults wrapper, file management, date formatting, locale handling, and Swift extensions.
  </Accordion>
  <Accordion title="PrismIntelligence — ML & AI">
    Unified ML client for on-device (CoreML/CreateML), Apple Intelligence (FoundationModels), and remote LLMs. Text and tabular classifiers, NLP (sentiment analysis, entity extraction), RAG pipeline with embedding store, model lifecycle management, and structured output parsing.
  </Accordion>
  <Accordion title="PrismCapabilities — Platform APIs">
    Native wrappers for Apple platform capabilities: biometric auth (Face ID, Touch ID, Optic ID), Sign in with Apple, location tracking, geofencing, camera capture, HealthKit, StoreKit (in-app purchases), CloudKit, WidgetKit, push notifications, App Intents, Bluetooth, NFC, MultipeerConnectivity, and more.
  </Accordion>
  <Accordion title="PrismVideo — Video Processing">
    Video metadata entity, actor-based video downloader with progress streaming via AsyncStream, resolution detection (4K/1080p/720p/SD), and AVFoundation export session management.
  </Accordion>
  <Accordion title="Prism — Umbrella Module">
    Single import that re-exports all client-side modules: PrismArchitecture, PrismCapabilities, PrismFoundation, PrismIntelligence, PrismNetwork, PrismUI, and PrismVideo. Use `import Prism` for full access.
  </Accordion>
  <Accordion title="PrismPreview — Preview App">
    A SwiftUI preview app target for developing and testing Prism components during development.
  </Accordion>
</AccordionGroup>

## A Real Example

Here's a realistic API — a task manager with database, authentication, and validation:

```swift title="Task Manager API"
import PrismServer

let server = PrismHTTPServer(port: 8080)
let db = try PrismDatabase(path: "tasks.db")

// Run migrations
let migrator = PrismMigrator(database: db)
try await migrator.migrate([
    PrismMigration(
        name: "create_tasks",
        up: "CREATE TABLE tasks (id INTEGER PRIMARY KEY, title TEXT NOT NULL, done INTEGER DEFAULT 0)",
        down: "DROP TABLE tasks"
    )
])

// Global middleware
await server.use(PrismCORSMiddleware())
await server.use(PrismLoggingMiddleware())
await server.use(PrismErrorMiddleware(includeStackTrace: true))

// List tasks
await server.get("/tasks") { request in
    let rows = try await db.query("SELECT * FROM tasks")
    let tasks = rows.map { row in
        ["id": row["id"], "title": row["title"], "done": row["done"]]
    }
    return .json(tasks)
}

// Create task with validation
await server.post("/tasks") { request in
    let result = request.validateJSON { v in
        v.field("title", .required, .minLength(1), .maxLength(200))
    }
    if let error = result.errorResponse() { return error }

    let title = request.formData["title"] ?? ""
    let changes = try await db.execute(
        "INSERT INTO tasks (title) VALUES (?)",
        parameters: [.text(title)]
    )
    return .json(["created": true], status: .created)
}

// Toggle task completion
await server.patch("/tasks/:id") { request in
    let id = request.parameters["id"]!
    try await db.execute(
        "UPDATE tasks SET done = NOT done WHERE id = ?",
        parameters: [.text(id)]
    )
    return .json(["updated": true])
}

try await server.start()
```
