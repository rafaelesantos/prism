Processamento de ações e produção de novo estado com `PrismReducer` e `PrismReduce`.

## Visão Geral

O reducer é a função pura que transforma estado em resposta a ações. No Prism, `PrismReducer` é o protocolo e `PrismReduce` é a implementação closure-based.

### PrismReducer (Protocolo)

Conforme `PrismReducer` para criar reducers tipados:

```swift
import PrismArchitecture

struct CounterReducer: PrismReducer {
    func reduce(into state: inout CounterState, action: CounterAction) -> PrismEffect<CounterAction> {
        switch action {
        case .increment:
            state.count += 1
            return .none
        case .decrement:
            state.count -= 1
            return .none
        case .reset:
            state.count = 0
            return .none
        }
    }
}
```

### PrismReduce (Closure)

Para reducers simples, use `PrismReduce` com closure:

```swift
let reducer = PrismReduce { (state: inout CounterState, action: CounterAction) in
    switch action {
    case .increment:
        state.count += 1
        return .none
    case .decrement:
        state.count -= 1
        return .none
    }
}
```

### Composição de Reducers

Combine múltiplos reducers em um:

```swift
let combined = PrismReduce.combine(
    counterReducer,
    userReducer,
    settingsReducer
)
```

### Retornando Effects

O reducer pode retornar `PrismEffect` para disparar efeitos assíncronos:

```swift
let reducer = PrismReduce { (state: inout AppState, action: AppAction) in
    switch action {
    case .loadUsers:
        state.isLoading = true
        return .run { send in
            let users = await fetchUsers()
            send(.usersLoaded(users))
        }

    case .usersLoaded(let users):
        state.users = users
        state.isLoading = false
        return .none
    }
}
```

### Erase e Handling

Converte qualquer `PrismReducer` para `PrismReduce` ou combina com middleware:

```swift
let erased = myReducer.eraseToReduce()
let withMiddleware = myReducer.handling(with: myMiddleware)
```
